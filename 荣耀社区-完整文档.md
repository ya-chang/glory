## 一、项目概述

### 1.1 是什么

基于 teamglory.netlify.app（全职高手同人站）扩展的社区论坛。三个核心板块：

```
💬 讨论区    — 关于全职高手的一切讨论
👤 选手百科  — 原作选手资料（只读，管理员维护）
🧑‍🎨 OC 角色中心 — 用户创建自己的原创角色
```

### 1.2 技术方案

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | 纯 HTML/CSS/JS | 和现有网站一致，手机可编辑 |
| 后端 | Firebase Firestore | 免费、无需服务器、自带实时同步 |
| 认证 | Firebase Auth | 零成本 |
| 托管 | Netlify（已有） | 直接加页面就行 |

### 1.3 手机开发

- 不需要构建工具，直接写原生 HTML/JS
- 编辑文件 → 推 GitHub → Netlify 自动部署
- 每次只改一个文件，改完刷新看效果

### OC 角色中心 — 关键规则

- **必填字段：** 角色名、游戏ID、年龄、性别、职业、武器、战斗风格、标志技能、战队、职位
- **选填字段：** 头像、立绘、身高、生日、外貌描述、性格描述、背景故事、人际关系、自创技能
- **头像规则：** 未上传 → 显示游戏ID文字占位
- **草稿箱：** 每步可保存，打开自动恢复，提交后清除

### 数据存储现状

| 数据 | 存储 | 说明 |

| 讨论区帖子 | localStorage | 含置顶、标签、点赞计数 |
| 讨论区回复 | localStorage | 支持楼中楼、@提及 |
| 讨论区点赞 | localStorage | 帖子/回复均可点赞 |
| 签到数据 | localStorage | 签到日期、连续天数 |
| 选手/战队 | JS 文件 | 写死在代码里，所有人可见 |

### 安全规则要点

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 用户：只读公开，只改自己
    match /users/{userId} {
      allow read: if true;
      allow write: if request.auth.uid == userId;
    }
    // 帖子：登录可发，只改自己的
    match /posts/{postId} {
      allow read: if true;
      allow create: if request.auth != null;
      allow update, delete: if request.auth.uid == resource.data.authorId;
    }
    // 评论：同上
    match /replies/{replyId} {
      allow read: if true;
      allow create: if request.auth != null;
      allow update, delete: if request.auth.uid == resource.data.authorId;
    }
    // 选手：所有人只读，仅管理员可写
    match /players/{playerId} {
      allow read: if true;
      allow write: if request.auth != null && get(/users/$(request.auth.uid)).data.isAdmin == true;
    }
    // OC：登录可创建，只改自己的
    match /ocs/{ocId} {
      allow read: if true;
      allow create: if request.auth != null;
      allow update, delete: if request.auth.uid == resource.data.creatorId;
    }
  }
}
```
